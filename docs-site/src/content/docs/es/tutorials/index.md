Cuatro casos de estudio, un mismo arco — el caso → los datos → la formulación del
problema → features → el run → la evaluación — modelados sobre el tutorial Dirty Duck
de DSSG triage y extendidos con las superficies que triage-pg agrega (dashboard,
fairness y audition dentro de PostgreSQL, evaluaciones de subconjuntos, monitoreo,
survival).

| Si estás… | Empieza con | Aprenderás |
| --- | --- | --- |
| verificando que tu instalación funciona | [**Dirty Duckling**](/triage-pg/es/tutorials/dirtyduckling/) | una prueba de humo pass/fail + un recorrido de 5 minutos por el dashboard |
| aprendiendo ML temporal sobre datos de política | [**DirtyDuck**](/triage-pg/es/tutorials/dirtyduck/) | el caso de estudio completo — alerta temprana vs priorización de recursos sobre los mismos datos |
| ejecutando algo con forma de producción | [**Chicago 311**](/triage-pg/es/tutorials/chicago311/) | auditoría de fairness, evaluaciones de subconjuntos, monitoreo, survival |
| metiéndote a fondo en la ingeniería de features | [**DonorsChoose**](/triage-pg/es/tutorials/donorschoose/) | Deep Feature Synthesis sobre un grafo de entidades multi-flujo |

Léelos en orden para el curso completo — la prueba de humo, los dos regímenes de
observación sobre un mismo dataset, las superficies de producción, los tres regímenes
de señal — o salta a la fila que te corresponda. El
[resumen en una página](https://ccd-ia.github.io/triage-pg/onboarding.html) es el mapa
del sistema de un vistazo, y los README de runbook en el
[repositorio](https://github.com/ccd-ia/triage-pg) siguen siendo las referencias
concisas de comandos.
